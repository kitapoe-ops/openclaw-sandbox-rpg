## 📋 Lessons Learned

1. **Inbound message content must be sanitized before any disk write.** This incident triggered a new rule (added below).
2. **Subagent briefs should explicitly mention "DO NOT echo any JWT / token / credential-looking strings from inbound context to disk."** This will be added to the M2 hand-off template going forward.
3. **M2 hand-off saved us here** — subagent's disk work was 100% token-free because the brief did not require the token, and subagent does not have access to the parent context's inbound payload.

## 🔒 New Hard Rule (added 2026-06-06 01:20 GMT+8)

> **Inbound credential strings (JWT, base64-encoded tokens, API keys, passwords) are NEVER:**
> - Echoed to stdout or stderr
> - Written to disk, log, or commit
> - Included in any doc, script, or summary
>
> **If an inbound message contains a credential-like string:**
> - Treat it as compromised by default
> - Quarantine (do not process)
> - Notify the user immediately
> - Recommend revocation
> - **Never assume the inbound content is safe to write or echo**
